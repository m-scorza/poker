## Confirmed Findings

#### Finding 1: Bundle weight regressing — eager shell now over 500 KB, lazy-PDF still unfixed

- **Status:** Recurring (abyss F11/F13, X-ray P4) / **Worsened**
- **Priority:** High
- **Evidence:** Build output: `index-*.js` 505.72 KB (163.8 KB gzip) vs
  367.1 KB on 2026-07-03; Vite emits its >500 KB warning. `SessionsPage`
  463.9 KB and `CareerPage` 443.8 KB (both up slightly). Grep of the shell
  chunk shows `framer-motion` and `dexie` present, `gsap`/`recharts` absent.
  `Layout.tsx:3` imports `CommandPalette`, which imports framer-motion —
  the 2026-07-03 build still had framer-motion in a separate lazy proxy
  chunk (118 KB). `SessionsPage.tsx:11` still statically imports
  `exportSessionsPDF` (`pdfExport.ts` statically imports jspdf/autotable).
- **Affected files/modules:** `src/components/layout/Layout.tsx`,
  `src/components/shared/CommandPalette.tsx`, `src/utils/pdfExport.ts`,
  `src/pages/SessionsPage.tsx`, `src/pages/CareerPage.tsx`.
- **Graphify signal:** n/a (no index); found via build output + chunk grep.
- **Direct code confirmation:** yes (imports read, chunks probed).
- **Why it matters:** every visit pays 505 KB before any route renders, in a
  local-first PWA whose pitch includes being lean; the regression happened
  silently in a feature merge, which is exactly how it will keep growing.
- **Recommended action:** one Wave 3 efficiency PR: (a) dynamic-import the
  PDF stack inside the export handler (F11), (b) load CommandPalette's
  motion code lazily (`LazyMotion`/`domAnimation` with `m.` components, or
  dynamic-import the palette on first Cmd+K), (c) optionally add a CI bundle
  budget so the next regression fails loudly.
