## Recommended cleanup rules going forward

1. If a doc is an active command/control doc, put it in `docs/agents/` and keep it short.
2. If a doc is a dated intent snapshot, keep it in `docs/plans/` and add a status banner when it becomes stale.

4. If a doc is design-only, keep it in `docs/design/` and do not let it drive backend tasks unless explicitly activated.
