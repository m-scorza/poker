## Status convention

Every report (except this README) begins with YAML frontmatter:

```md
---
status: open | in_progress | resolved | superseded
date: YYYY-MM-DD
related: ['#74', 'docs/product/STATUS.md']            # optional: what closed it / context
superseded_by: 2026-06-12-principal-engineer-audit.md  # required iff status: superseded
---
# <Report title>
```

| status | meaning | location |
|---|---|---|
| `open` | has unfinished action items — **still to look at** | `docs/reports/` |
| `in_progress` | actively being worked right now | `docs/reports/` |
| `resolved` | every action item is done | `docs/reports/archive/` |
| `superseded` | replaced by a newer report (see `superseded_by`) | `docs/reports/archive/` |

The index display title comes from each report's first `# ` heading — there is no
separate `title:` field, so it can't drift. A report with **no `status`** is a hard
error: `docs:check` fails until it is classified.

## Directory layout

- `docs/reports/` — **active** reports (`open` / `in_progress`) + this README.
- `docs/reports/archive/` — **done** reports (`resolved` / `superseded`).

## How active reports reach agents

`scripts/surface-open-reports.ts` runs as a **SessionStart hook**: at the start of
every Claude Code session its stdout is injected into context so no agent starts
work without seeing what's active. It prints nothing when nothing needs attention.

A report is surfaced only when **both** hold: its status is `open` or `in_progress`,
**and** it has an `## Open items` (or `## Recommended next actions`) section. Reports
without such a section (e.g. a reference appendix) stay silent — the index still
lists them. The hook scans only this active top level, so `archive/` never surfaces.

### Hook configuration

Configured in the tracked `.claude/settings.json` as this `hooks` block (alongside
`permissions`):

```json
"hooks": {
  "SessionStart": [
    {
      "matcher": "startup|resume|clear|compact",
      "hooks": [
        { "type": "command",
          "command": "npx tsx \"${CLAUDE_PROJECT_DIR}/scripts/surface-open-reports.ts\"" }
      ]
    }
  ]
}
```

Preview what it would inject any time: `npx tsx scripts/surface-open-reports.ts`.

## Lifecycle

1. A review lands in `docs/reports/` as `YYYY-MM-DD-<name>.md` with `status: open`
   and an `## Open items` section.
2. As items close (by PRs), trim `## Open items` and add the PR to `related:`.
3. When the last item is done, set `status: resolved` (or `superseded` +
   `superseded_by:`), `git mv` the file into `archive/`, and run
   `npm run docs:update` to refresh the index.

This keeps "what's left" in front of every session until it is genuinely done, and
the enforced index keeps the completed/superseded record honest — the exact drift
(a finished report still flagged, or a live finding silently buried) this is meant
to prevent.
