# Competitor and Market Research

Scope: obvious incumbents, obscure poker apps, Instagram-ad tools, coaches selling productized training, trackers, solvers, trainers, bankroll/career tools, and adjacent analytics products.

## Matrix fields

| Field | Meaning |
|---|---|
| Product | Name and URL. |
| Audience | Grinder, beginner, coach, MTT player, cash player, mobile casual, etc. |
| Core promise | What the product says it solves. |
| Pricing / trial | Public pricing, free tier, trial, upsell. |
| Platform | Web, desktop, mobile, local, cloud. |
| Import/data model | Hand-history import, manual spots, provider DB, solver tree, trainer-only. |
| Core workflow | Analyze, drill, review, track, coach, community, solver. |
| Differentiators | What they do unusually well. |
| Trust signals | Solver-backed, coach-backed, database-backed, testimonials, privacy claims. |
| Weaknesses | Gaps, confusing claims, pricing friction, unsupported MTT contexts. |
| Opportunity for us | Product wedge or avoided trap. |

## Research rule

Every major product move should be compared against competitors before implementation, but do not let competitor breadth delay small validated product increments.
