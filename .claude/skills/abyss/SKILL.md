# /abyss — The Audit Ritual

"The most thorough scan possible — even ugly code shall be marked and vanish
from existence later." Planning only: every finding is marked with evidence;
**nothing is fixed in this session**. Execution happens later via the wave
plan, each wave a gated PR.

## Method (match the 2026-07-01 abyss audit's bar)

1. **Baseline the gates first.** `npx tsc -b --pretty false`, `npm test`,
   `npm run build`, `npm run docs:check`. A broken gate is automatically
   finding F1 — nothing else outranks a red baseline.
2. **Tooling sweep, each finding manually verified** (a tool hit is a lead,
   not a finding): `npx madge --circular src`, `npx jscpd src`, `npx knip`
   for dead files/exports/deps; hygiene greps for `any`, `@ts-ignore`,
   `eslint-disable`, `TODO/FIXME/HACK`, stray `console.`, hardcoded hex
   colors (token bypasses); production build with chunk sizes.
3. **Full reads of the largest/most central files** (store, upload, replay,
   parsers, top pages) plus targeted reads wherever the tools pointed.
   Severity axes, in priority order: **correctness → dead code → efficiency →
   beauty**. Ugly-but-correct is still a finding; wrong is always first.
4. **Number every finding** (F1, F2, …) with file:line evidence, severity,
   and the cheapest honest fix. No vibes-based findings — if it wasn't
   verified against source, it doesn't get a number.

## Output — a dated report, nothing else

Write `docs/reports/<YYYY-MM-DD>-abyss-audit.md` (suffix `-2`, `-3` if the
date collides) in the house format:

```
---
status: open
date: <YYYY-MM-DD>
related: [<relevant plans/reports>]
---
```

- Open with **"0. Verdict first"** — the honest one-paragraph state of the
  codebase before any detail.
- Findings grouped by severity axis, each numbered with evidence.
- Close with **"## Open items"**: a wave plan, cheapest-truth-first —
  correctness quickies → dead code → efficiency → beauty — each wave sized as
  one gated PR.
- Add the report to the `docs/reports/README.md` index (autogenerated /
  `docs:check`-enforced — run `npm run docs:update` if that's what maintains
  it) so the SessionStart hook surfaces it.

## Rules

- Compare against the previous abyss/health reports in `docs/reports/` and
  `docs/reports/archive/` — regressions on previously-fixed findings get
  called out explicitly (this repo has a history of stamps going stale).
- Don't re-litigate parked decisions (villain auto-archetypes, derived-stats
  cache — exonerated) unless new evidence appears; the ROADMAP's non-goals
  list is binding.
- Long runs: this pairs with the global `overnight` skill for unattended
  execution — same exit criteria, findings appended to the report file.
